# Wonderblob UX Pass ‚Äî Implementation Plan

> Execute in order; each chunk builds, type-checks (`npm run check`), and commits independently. Search is deferred (stretch). Frontend-only unless noted.

**Goal:** Make the app's already-built power discoverable and add the missing UX: icons, context menus, sorting, tree view, settings, resizable panes ‚Äî per user feedback 2026-06-08.

**Foundational decisions:**
- Inline-SVG `Icon.svelte` (Lucide-style paths, no dependency).
- Reusable `ContextMenu.svelte` (cursor-positioned, keyboard-dismiss, outside-click close).
- Right-click is currently globally prevented in `+layout.svelte`; rows that have a context menu will `preventDefault` + open ours.

**Key files:** `src/lib/components/{Icon,ContextMenu}.svelte` (new), `FileList.svelte`, `BookmarkList.svelte`, `TransfersPanel.svelte`, `src/routes/+page.svelte`, `+layout.svelte`, new `Settings.svelte`.

## Chunk 5 ‚Äî Sorting

## Chunk 6 ‚Äî Resizable sidebar + columns

- File columns: draggable resize handles; a header context menu to show/hide optional columns (e.g., Modified, Kind). Persist column widths/visibility.
